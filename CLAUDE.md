# How Safe Is AI?

Daily-updated interactive visualization tracking CVEs affecting AI systems. Vulnerabilities appear as biological colonies growing in petri dishes. Static site on GitHub Pages, data refreshed nightly via GitHub Actions.

Full PRD and build guide: `docs/CLAUDE.md` and `docs/BUILD-INSTRUCTIONS.md`.

---

## Commands

```bash
npm run dev                  # dev server → localhost:5173 (auto-reloads)
npm run build                # production build → dist/
npm run preview              # preview production build locally
python scripts/fetch-cves.py # run data pipeline manually
```

## Architecture

```
src/
├── main.jsx                        # entry point
├── App.jsx                         # root: loads data, manages selected dish state
├── components/
│   ├── IntroSection.jsx            # hero text + how-to-read guide + headline stats bar
│   ├── LabBench.jsx                # 3×3 CSS grid with lab bench texture
│   ├── PetriDish.jsx               # individual dish: SVG filter layer + Canvas layer
│   ├── LabNotes.jsx                # notes panel (shown on dish click, hover-highlights dish)
│   ├── Footer.jsx                  # methodology + data source credits
│   └── dishes/                     # one file per dish, each exports a draw(canvas, data) function
│       ├── PromptInjection.js      # radiating tendrils from center
│       ├── SupplyChain.js          # Poisson-disc scattered multi-color colonies
│       └── DefaultPattern.js       # organic metaball blobs — fallback for all other dishes
├── utils/
│   ├── rendering.js                # shared canvas helpers (metaballs, breathing animation, particles)
│   └── filters.js                  # SVG filter generators (feTurbulence, feDisplacementMap, etc.)
├── config/
│   └── categories.json             # category definitions, colors, keywords, product mappings
├── data/
│   └── sampleData.js               # sample data used during development (Phases 1–5)
└── styles/
    └── global.css

data/                               # GENERATED by pipeline — committed to repo, read at runtime
├── cves.json
└── stats.json

scripts/
└── fetch-cves.py                   # queries NVD + GitHub Advisory DB → outputs data/cves.json

.github/workflows/
└── update-data.yml                 # runs fetch-cves.py daily at 00:00 UTC, commits output
```

**Data flow:** pipeline writes `data/cves.json` → committed to repo → frontend fetches at page load → passed as props to `LabBench` → each `PetriDish` receives its category slice → dish `draw()` function renders to canvas.

---

## Hard Constraints

- **No runtime API calls.** The frontend reads only committed JSON files (`data/cves.json`, `data/stats.json`). All external requests happen in the Python pipeline at build time.
- **Static site only.** No SSR, no server, no backend. Must deploy to GitHub Pages as-is.
- **Categories are config-driven.** Read from `src/config/categories.json`. Never hardcode category IDs, names, colors, or keywords in component logic.
- **One category per CVE.** Priority-ordered keyword matching in the pipeline ensures no CVE appears in multiple dishes.
- **Performance.** 9 canvas animations run simultaneously. Always use `requestAnimationFrame`. Use offscreen canvases and throttle redraws. Never block the main thread.

---

## Code Conventions

- Components: PascalCase `.jsx` files in `src/components/`
- Dish draw functions: one `.js` file per dish in `src/components/dishes/`, each exports `draw(canvas, categoryData, options)` — no React, pure canvas
- New dish patterns: add a file in `dishes/`, import and register in `PetriDish.jsx` — do not modify other dish files
- Shared canvas utilities live in `src/utils/rendering.js` — reuse these rather than reimplementing (metaball, breathing pulse, particle system)
- SVG filters are generated programmatically via `src/utils/filters.js` — do not write filter markup inline

## Design System

**Aesthetic:** Bright scientific editorial. Warm lab bench. NOT dark cyberpunk.

| Token | Value |
|---|---|
| Page background | `#f5f2eb` (warm cream) |
| Intro typography | Serif or slab-serif |
| Lab notes / data | Monospace |
| Dish glass rim | Light gradient + subtle reflection arc |
| Bench texture | Faint grain + soft shadows under dishes |

**Dish rendering (SVG + Canvas hybrid):**
- SVG filter layer: `feTurbulence` (agar grain) + `feDisplacementMap` (organic edges) + `feGaussianBlur` (color bleed) + `feColorMatrix` (per-category tint)
- Canvas layer: metaball-style colony blobs drawn per category's `draw()` function

**Per-dish colors:**

| Dish | Agar tint | Colony colors |
|---|---|---|
| Prompt Injection | Pale pink | Hot pink → magenta |
| Improper Output Handling | Pale coral | Bright blue on red-orange |
| Model Poisoning | Pale yellow | Dark brown/amber, speckled |
| Data Leakage | Pale cream | Deep blue rings, yellow rim |
| Denial of Service | Pale green | Vivid green + blue, dense |
| Supply Chain | Dark/black | Multi-colored scattered |
| Model Theft | Pale teal | Deep teal, white veins |
| Agentic Autonomy | Pale amber | Orange-yellow clusters |
| Vector / Embedding | Pale green | Green lace/web patterns |

**Data → visual mapping:**

| Data field | Visual encoding |
|---|---|
| CVE count per product | Colony blob size |
| Severity | Color intensity (critical=vivid, low=faint) |
| Number of products | Number of distinct colonies |
| `exploited: true` | Pulsing bioluminescent glow |
| `patched: true` | Faded/desaturated colony |
| `patched: false` | Vivid/bright colony |
| QoQ trend ↑ | Colonies spread toward edges |
| QoQ trend ↓ | Colonies pull toward center |

**Severity → spatial zones (used for hover-highlight in LabNotes):**
- Critical → center/core
- High → mid-ring
- Medium → outer ring
- Low → faint edges

## Data Schema

### data/cves.json
```
{
  lastUpdated: string (ISO 8601),
  summary: {
    totalCVEs, critical, high, medium, low,
    exploited, patchedPercent, quarterOverQuarterChange
  },
  cves: [{
    id, category, product, cvss,
    severity ("critical"|"high"|"medium"|"low"),
    exploited (bool), patched (bool),
    published (YYYY-MM-DD), description
  }]
}
```

Category IDs match keys in `src/config/categories.json`.

## Interactions

| User action | Result |
|---|---|
| Idle | Colonies breathe (~4s pulse), particles drift, exploited CVEs glow |
| Hover dish | Dish brightens, tooltip shows category name + CVE count |
| Click dish | LabNotes panel appears beside dish; other dishes fade + blur |
| Hover severity row in notes | Corresponding spatial zone in dish highlights |
| Hover "Patched" in notes | Faded colonies highlight |
| Hover "Unpatched" in notes | Vivid colonies highlight |
| Click outside / click dish again | Notes dismiss, all dishes return to normal |
